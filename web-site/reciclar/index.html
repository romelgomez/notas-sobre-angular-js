Conceptos Generales

Este documento ofrece una visión general de los principales componentes de angular y cómo trabajan juntos. Estos son:

  • startup (inicio) - Hola mundo.
  • runtime (tiempo de ejecución) - Revisión general del tiempo de ejecución de angular.
  • scope (ámbito) - El pegamento entre la vista y el controlador.
  • controller (controlador) - El comportamiento de la aplicación.
  • model (modelo) - Los datos de la aplicación.
  • view (vista) - Lo que el usuario ve.
  • directives (directivas) - Amplia el vocabulario HTML.
  • filters (filtros) - Formatea la data para el usuario.
  • injector (injector) - Ensambla su aplicación.
  • module (módulo) - Configura la inyección.
  • $ - Espacio de nombre o namespace de AngularJS.

Startup (Inicio)

Así es como hacemos rodar la pelota (observe el siguiente diagrama y ejemplo):

  1. El buscador carga el HTML y lo analiza en un DOM.
  2. El navegador carga el script angular.js.
  3. Angular espera por el evento DOMContentLoaded.
  4. Angular busca la directiva ng-app, la cual define el ámbito o límites de la aplicación.
  5. El modulo especificado en ng-app (si existe) es usado para configurar el $injector.
  6. El $injector es usado para crear el servicio $copile como también $rootScope.
  7. El servicio $compile es usado para compilar el DOM y enlazarlo con $rootScope.
  8. La directiva ng-init asigna la palabra Mundo al nombre de la propiedad en el scope.
  9. El {{nombre}} interpola la expresión en: Hola Mundo!

Código fuente:

  • index.html
<!doctype html>
<html ng-app>
  <head>
    <script src="http://code.angularjs.org/1.0.5/angular.min.js"></script>
  </head>
  <body>
    <p ng-init=" name='Mundo' ">Hola {{nombre}}!</p>
  </body>
</html>

Runtime (Tiempo de ejecución)

El diagrama y el ejemplo a continuación describen cómo angular interactúa con el evento repetitivo del navegador.

  1. El evento repetitivo (event-loop) del navegador espera por el arribo de un evento. Un evento es una interacción del usuario, un evento agendado, o un evento de la red (una respuesta proveniente del servidor).
  2. Las retrollamadas de eventos son ejecutadas, Se ingresa el el contexto JavaScript, La retrollamada puede modificar la estructura DOM.
  3. Una vez que la retrollamada es ejecutada, el navegador deja el contexto JavaScript y rehace la vista basado en los cambios del DOM.

Angular modifica el flujo normal de JavaScript proporcionado su propia forma para procesar el evento repetitivo. Dividiendo el código JavaScript en el contexto de ejecución clásico y angular. Sólo las operaciones que son aplicadas en el contexto de ejecución de angular se beneficiarán del enlace de datos (data-binding), manejo de excepciones (exception handling), observación de propiedad (property watching), etc... También puede usar $apply() para entrar en el contexto de ejecución de angular desde JavaScript. Tenga en cuenta que en la mayoría de los lugares (controladores, servicios) $apply ya ha sido llamado para usted por la directiva que maneja el evento. Una llamada explícita a $apply es necesaria sólo cuando se implementen retrollamadas personalizadas o cuando trabajes con retrollamadas en librerías de terceros.

  1. Entra contexto de ejecución Angular llamando scope.$apply(estimuloFn). Donde estimuloFn es el trabajo que se desea realizar en el contexto de ejecución angular.
  2. Angular ejecuta estimuloFn(), que normalmente modifica el estado de aplicación.
  3. Angular entra en el bucle $digest. El bucle se compone de dos bucles más pequeños que procesan la cola $evalAsync y la lista $watch. El bucle $digest sigue iterando hasta que el modelo se estabiliza, lo que significa que la cola $evalAsync esta vacia y la lista $watch no detecta ningún cambio.
  4. La cola $evalAsync es utilizada para programar el trabajo que tiene que ocurrir fuera del marco de pila actual, pero antes que el navegador muestre la vista. esto es usualmente realizado con setTimeout(0), pero el enfoque setTimeout(0) es lento y puede causar parpadeo en la vista ya que el navegador hace la vista luego de cada evento.
  5. La lista $watch es un conjunto de expresiones que pueden haber cambiado desde la última iteración. Si se detecta un cambio entonces la funcion $watch es llamada la cual suele actualizar el DOM con un nuevo valor.
  6. Una vez que el bucle $digest finaliza, la ejecución deja el contexto Angular y JavaScript. A esto le sigue el navegador re-haciendo el DOM para reflejar algún cambio.

Aquí está la explicación de cómo el ejemplo Hola Mundo logra el efecto de enlace de datos cuando el usuario introduce texto en el campo de texto.

  1. Durante la fase de compilación:
    1. El ng-model y la directiva input establecen un detector keydown sobre el control <input>.
    2. La interpolación {{nombre}} establece un $watch para ser notificado del los cambios de nombre.
  2. Durante la fase de ejecución:
    1. Presionando 'X' tecla, causa que el navegador emita un evento keydown sobre el control de entrada.
    2. La directiva input captura los cambios de valor de los inputs (campos) y llama $apply("nombre = 'X';") para actualizar el modelo de la aplicación dentro del contexto de Angular.
    3. Angular aplica el nombre = ''; al modelo.
    4. El bucle $digest comienza.
    5. La lista $watch detecta un cambio en la propiedad nombre y notifica la interpolación {{nombre}}, que a su vez actualiza el DOM.
    6. Abandona el contexto de ejecución de Angular, que a su vez sale de el evento keydown y con ello el contexto de ejecución de de JavaScript.
    7. El navegador rehace la vista con el texto actualizado.

Controller (Controlador)

Un controlador es el código detrás de la vista. Su trabajo es construir el modelo y publicarlo en la vista junto con las retrollamadas. La vista es una proyección de elámbitoo en la plantilla (el HTML). El ámbito es el pegamento que reúne el modelo a la vista y envía los eventos al controlador.

La separación del controlador y la vista es importante porque:

  • El controlador está escrito en javascript. javascript es imperativo. Imperativo es una buena opción para especificar el comportamiento de la aplicación. El controlador no debe contener ninguna información de representación (referencias DOM o fragmentos HTML).
  • La plantilla de vista está escrita en HTML. HTML es declarativo, Declarativa es una buena opción para especificar la interfaz de usuario. La vista no debería contener ningún comportamiento.
  • Puesto que el controlador no tiene conocimiento de la vista, Puede haber muchas vistas para el mismo controlador. Esto es importante para rehacer las vistas para los diferentes dispositivos por ejemplo teléfonos inteligentes y escritorios además de la capacidad de prueba.

Código fuente:

  • index.html
  • script.js
<!doctype html>
<html ng-app>
  <head>
    <script src="http://code.angularjs.org/1.0.5/angular.min.js"></script>
    <script src="script.js"></script>
  </head>
  <body>
    <div ng-controller="MiCtrl">
      Hola {{nombre}}!
      <button ng-click="metodo()">
      	OK
      </button>
    </div>
  </body>
</html>
// Controlador 
function MiCtrl($scope) {
	
	// Metodo
	$scope.metodo = function() {
		$scope.nombre = 'OK';
	}
 
	// Propiedad
	$scope.nombre = 'Mundo';

}

Model (Modelo)

El modelo es la data que es usada funcionada con la plantilla para producir la vista, para ser capaz de mostrar el modelo en la vista, el modelo tiene que ser capaz de ser referenciado desde el ámbito o scope. A diferencia de muchos otros marcos angular no crea restricciones o requisitos en el modelo. No hay clases para heredar o especiales métodos ancestros para acceder o cambiar el modelo. El modelo puede ser primitivo, objeto hash, o un objeto completo. En resumen, el modelo es un simple objeto JavaScript.

View (Vista)

La vista es lo que usuario ve. La vista comienza su vida como una plantilla, se funciona con el modelo y finalmente se muestra en el DOM. Angular tiene un enfoque muy diferente para mostrar la vista en comparación con la mayoria de sistema de plantillas.

  • Otros - La mayoría de los sistemas de plantillas comienzan como una cadena HTML con un especial marcado. A menudo, el formato de plantilla rompe la sintaxis HTML lo que significa que la plantilla no se puede editar con el editor HTML. La cadena es analizada por el motor de plantillas, y se combina con los datos. El resultado de la fusión es una cadena HTML. La cadena HTML se escribe en el navegador utilizando el innerHTML, que causa que el navegador muestre el HTML. Cuando el modelo cambia todo el proceso debe ser repetido. La granularidad de la plantilla es la granularidad de las actualizaciones de DOM. La clave aquí es que el sistema de plantillas manipula cadenas.
  • Angular - Angular es diferente, ya que su sistema de plantillas funciona en objetos DOM no en cadenas. La plantilla se sigue escribiendo en una cadena HTML, pero es HTML (no HTML con un especial marcado) El navegador analiza el código HTML en el DOM, y el DOM se convierte en la entrada del motor plantilla conocida como compilador (compiler). El compilador busca directivas a la vez que establece observadores en el modelo, El resultado es una continua actualización de la vista que no necesita de un modelo de sistema de plantillas que vuelva a funcionar el HTML con un especial marcado. Su modelo se convierte en la única fuente verdadera para su vista.

Filters (Filtros)

Los filtros realizar la transformación de los datos. Típicamente se utiliza en conjunción con la configuración regional para formatear los datos de salida en una configuración regional específica. Ellos siguen el espíritu de los filtros UNIX y usan una sintaxis similar | (tubo).

Código fuente:

  • index.html
<!doctype html>
<html ng-app>
<head>
	<script src="http://code.angularjs.org/1.0.5/angular.min.js"></script>
</head>
<body>
	<h5>Formateo de número:</h5>
	{{ 1234567890 | number }}

	<h5>Filtrado de Array:</h5>
	<div ng-init="list = ['Chrome', 'Safari', 'Firefox', 'IE'] ">
		<input ng-model="predicado"> {{ list | filter:predicado | json }}
	</div>
</body>
</html>

Modules and the Injector (Módulos y el Inyector)

El inyector es un servicio de localización. Hay un solo inyector por una aplicación angular. El injector provee una forma para mirar un objeto instanciado por su nombre. El injector mantiene una caché interna de todo los objetos a fin de que las redundante solicitudes para obtener el mismo objeto resulte en la misma instancia. Si el objeto no existe, Entonces el injector solicita al la fábrica de instancias que crea una nueva instancia.

Un módulo es la forma de configurar la fábrica de instancias de Inyectores. conocido como un proveedor.


// Create a module
var myModule = angular.module('myModule', [])
 
// Configure the injector
myModule.factory('serviceA', function() {
  return {
    // instead of {}, put your object creation here
  };
});
 
// create an injector and configure it from 'myModule'
var $injector = angular.injector(['myModule']);
 
// retrieve an object from the injector by name
var serviceA = $injector.get('serviceA');
 
// always true because of instance cache
$injector.get('serviceA') === $injector.get('serviceA');

Pero la real magia del Inyector es que puede ser utilizado para llamar métodos y tipos de instancias. Esta característica sutil es la que permite los métodos y tipos de solicitar sus dependencias en lugar de tener que buscarlos.

// You write functions such as this one.
function doSomething(serviceA, serviceB) {
  // do something here.
}
 
// Angular provides the injector for your application
var $injector = ...;
 
///////////////////////////////////////////////
// the old-school way of getting dependencies.
var serviceA = $injector.get('serviceA');
var serviceB = $injector.get('serviceB');
 
// now call the function
doSomething(serviceA, serviceB);
 
///////////////////////////////////////////////
// the cool way of getting dependencies.
// the $injector will supply the arguments to the function automatically
$injector.invoke(doSomething); // This is how the framework calls your functions

Observe que lo único que necesitaba era escribir la función y la lista de dependencias en los argumentos de la función. Cuando angular llama la función, utiliza la solicitud para automáticamente llenar los argumentos de la función.

Examine el controlador ClockCtrl a continuación y observe como se enumeran las dependencias en el constructor. cuando el ng-controller instancias el controlador automáticamente provee las dependencias. no hay necesidad de crear dependencias, buscarlas o incluso conseguir una referencia al inyector.

Código fuente:

  • index.html
  • script.js
<!doctype html>
<html ng-app="timeExampleModule">
  <head>
    <script src="http://code.angularjs.org/1.0.5/angular.min.js"></script>
    <script src="script.js"></script>
  </head>
  <body>
    <div ng-controller="ClockCtrl">
      Current time is: {{ time.now }}
    </div>
  </body>
</html>
angular.module('timeExampleModule', []).
  // Declare new object called time,
  // which will be available for injection
  factory('time', function($timeout) {
    var time = {};
 
    (function tick() {
      time.now = new Date().toString();
      $timeout(tick, 1000);
    })();
    return time;
  });
 
// Notice that you can simply ask for time
// and it will be provided. No need to look for it.
function ClockCtrl($scope, time) {
  $scope.time = time;
}
